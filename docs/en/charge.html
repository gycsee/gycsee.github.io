主收费项目

1 新增收费项目

1.1 请求链接

  • <host>/api/vrp/v1/charge/addMasterCharge

1.2 请求类型

  • POST

1.3 请求参数

请求参数参数类型参数说明是否必填
chargeIdString收费项目ID是
chargeDescString收费项目描述是
chargeTypeString收费类型是
isDeviceSpecString是否基于车型收费是
lookupUnitString查找因子是
rateUnitString收费因子是
deptIdString部门ID否
branchIdString公司ID否

1.4 请求参数示例

{
  "branchId": "DEPPON",
  "chargeDesc": "整车价",
  "chargeId": "R10",
  "chargeType": "MANDANTORY",
  "deptId": "DEPPON-HZ",
  "isDeviceSpec": "Y",
  "lookupUnit": "DEVICE_TYPE",
  "rateUnit": "DEVICE_TYPE"
}

1.5 返回结果

返回参数参数类型参数说明
httpCodeIntegerhttp状态码
retCodeInnteger返回结果编码
retInfoString返回结果英文信息
retMsgString返回结果中文信息
chargeIdString收费项目ID
chargeDescString收费项目描述
chargeTypeString收费类型
isDeviceSpecString是否基于车型收费
lookupUnitString查找因子
rateUnitString收费因子
deptIdString部门ID
deptNameString部门名称
branchIdString公司ID
branchNameString公司名称
createDttLocalDateTime创建时间（yyyy-mm-dd hh:mm:ss）
createByString创建人
updateDttLocalDateTime最后更新时间（yyyy-mm-dd hh:mm:ss）
updateByString最后更新人

1.6 返回结果示例

{
  "data": {
    "branchId": "DEPPON",
    "branchName": "德邦物流",
    "chargeDesc": "整车价",
    "chargeId": "R10",
    "chargeType": "MANDANTORY",
    "createBy": "yuguanquan",
    "createDtt": "2018-06-27 18:42:44",
    "deptId": "DEPPON-HZ",
    “deptName”: "德邦杭州车队",
    "isDeviceSpec": "Y",
    "lookupUnit": "DEVICE_TYPE",
    "rateUnit": "DEVICE_TYPE",
    "updateBy": "yuguanquan",
    "updateDtt": "2018-06-27 18:42:44"
  },
  "httpCode": 200,
  "retCode": 0,
  "retInfo": "OPERATE_SUCCESS",
  "retMsg": "操作成功"
}

2 批量删除收费项目

2.1 请求链接

  • <host>/api/vrp/v1/charge/deleteBatchMasterCharge

2.2 请求类型

  • DELETE

2.3 请求参数

请求参数参数类型参数说明是否必填
chargeIdsList<String>收费项目ID列表是

2.4 请求参数示例

{
  "ids": [
    "R08","09"
  ]
}

2.5 返回结果

返回参数参数类型参数说明
httpCodeIntegerhttp状态码
retCodeInnteger返回结果编码
retInfoString返回结果英文信息
retMsgString返回结果中文信息
errorList<ErrorVO>删除失败信息列表
successList<String>删除成功的id列表

2.6 返回结果示例

{
  "data": {
    "error": [
      {
        "error": "收费项目ID不存在",
        "id": "09"
      }
    ],
    "success": [
      "R08"
    ]
  },
  "httpCode": 200,
  "retCode": 0,
  "retInfo": "OPERATE_SUCCESS",
  "retMsg": "操作成功"
}
