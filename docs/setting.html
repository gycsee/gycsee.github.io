# 系统配置维护

系统配置功能提供了系统级别的设置，包括公司部门设置、用户角色与权限配置、用户管理、全局设置等功能。 通过系统设置可以定义一些企业组织架构和相应的数据权限。以及一些全局的配置。

# 1. 公司

通过“系统设置>公司”可以查看并编辑公司信息。

# 1.1 公司详情（字段说明）

  • 通用信息

    • 公司代码：系统中公司的唯一代码，新增时公司代码不允许重复。
    • 公司描述：可增加公司的具体描述信息。
    • 优化器版本：可选项（非必填）；可下拉选择不同版本的优化器，若未选择则算法版本以全局设置中的选择为准，若已选择则会覆盖全局设置中的选项，作为最终采用的算法版本。

  • 创建信息

    • 创建人/创建时间/最后更新人/最后更新时间：由系统自动记录的当前数据创建或更新的具体信息，不可编辑。

# 2. 部门

通过“系统设置>公司”可以查看并编辑公司信息。

# 2.1 部门详情（字段说明）

  • 通用信息

    • 部门代码：系统中部门的唯一代码，新增时部门代码不允许重复。
    • 部门描述：可增加部门的具体描述信息。
    • 优化器版本：可选项（非必填）；可下拉选择不同版本的优化器，若未选择则算法版本以公司（或全局设置）页面中的选择为准，若已选择则会覆盖公司及全局设置页面中的选项，作为最终采用的算法版本。
    • 所属公司代码：下拉从已维护的公司列表中选择所属公司。
    • 所属公司代码：下拉从已维护的公司列表中选择所属公司。

  • 创建信息

    • 创建人/创建时间/最后更新人/最后更新时间：由系统自动记录的当前数据创建或更新的具体信息，不可编辑。

# 3. 用户

通过“系统设置>用户”可以查看并编辑公司信息。

# 3.1 查看与编辑

  • 查看已维护的用户账号信息

    • 可查看用户名、所属用户组、权限控制情况、客户、联系方式等多种信息

  • 编辑用户账号信息

    • 可手动输入用户代码、用户名、联系方式、所属公司及部门等用户账号的基本信息。
    • 可多选用户所属角色，根据角色控制菜单访问权限，多角色账户按所有角色的菜单权限总和分配。
    • 用户代码不允许重复。
    • 可分配多级数据权限，允许账户查看相应公司/部门的数据。
    • 可以配置用户的有效期，为空表示永久有效。

# 3.2 用户详情（字段说明）

  • 通用信息

    • 用户名：系统用户账号的唯一代码，命名时只能使用数字、字母或下划线。
    • 用户描述：可以增加对用户账号的详细描述。
    • 是否启用：可调整用户账号的启停用状态，已停用的用户不能登陆系统。
    • 邮箱：可手动输入用户的联系邮箱，存在邮箱地址格式校验。
    • 手机：可手动输入用户的联系手机号码，存在手机号格式校验。
    • 所属公司/部门代码：下拉选择用户所属的公司/部门代码。
    • 有效期：用户的有效期维护，空表示永久有效。
    • 所属角色：下拉选择用户的所属角色，可多选，同时选择多种角色时，权限取所有角色的并集。

  • 创建信息

    • 创建人/创建时间/最后更新人/最后更新时间：由系统自动记录的当前数据创建或更新的具体信息，不可编辑。

  • 数据权限

    • 系统中已维护的公司及部门数据名称。

# 3.3 用户数据权限

  • 系统内的相关数据，均包含以下层级

    • 部门级数据：指定公司，也指定部门的数据 。
    • 公司级数据：指定公司，不指定部门的数据。
    • 全局数据用：不指定公司，不指定部门的数据。

  • 数据权限

    • 业务数据必须是部门级数据，不同公司/部门之间的业务数据是隔离的。

    • 基础数据3个等级都可以存在。

    • 用户拥有到部门的数据权限，可以查看或修改：同公司同部门的部门级数据/同公司的公司级数据/全局数据。

    • 用户拥有多个部门数据的权限，可以查看多个部门的数据。

# 4. 用户角色

通过“系统设置>用户”可以查看并编辑用户角色信息。

# 4.1 查看与编辑

  • 可查看或编辑用户角色名称、类型、所属部门等信息。
  • 用户组代码不允许重复，可以为角色配置相应的菜单权限。

# 4.2 用户角色详情（字段说明）

  • 通用信息

    • 用户名：系统用户角色的唯一代码。
    • 角色描述：可以增加对用户角色的详细描述。
    • 是否启用：可调整用户账号的启停用状态，已停用的用户不能登陆系统。

  • 创建信息

  • 创建人/创建时间/最后更新人/最后更新时间：系统自动记录创建/更新信息，不可编辑。

  • 权限分配

    • 系统中所有模块及页面名，字面右边的“功能分配”字段存在超链接，可点击分配功能权限。

    • 可以配置相应的菜单权限是否分配给对应的用户角色

# 5. 全局设置

通过“系统设置>全局设置”可以进行系统基本配置，并查看已有配置详情。

# 5.1 进行系统基本配置

  • 系统目前支持定制化的算法版本器配置以及修改租户名称，配置距离引擎。

  • 算法版本器包括：表明系统支持的算法版本器，特定客户可以定制算法版本器。

    • 标准版算法
    • Sprit版算法

  • 距离引擎包括：

    • 经纬度距离：按经纬度直接算球面距离。
    • 高德：通过实时调取高德返回实际两点距离。
    • 杉数距离引擎：基于高德的杉数锚点模糊距离矩阵。
    • 杉数精确距离：调取高德获取两点实际距离并保存到后台，定期更新两点间的距离和行驶时间。

  • 可以配置运输地点页面的业务区域/限行区域的默认维护方式

    如果配置为系统解析，则在运输地点维护的时候，自动解析地址所属区域开关将会被打开，在保存运输地点时将会自动解析所属的业务区域/限行区域

    如果配置为手动解析，则需要人工在运输地点维护时候指定所属的业务/限行区域。

  • 可以配置当运输地点/区域的可用车型为空的时候算法如何处理

    当运输

  • 可以配置是否启用计费引擎进行优化与计算总费率。

# 6. 数据字典

数据字典用于维护和展示系统中大部分下拉框值的内容。包括系统关键字（不可修改）和用户自定义数据（可修改）

  • 可查看字段名，及其是否由系统提供。
  • 只可编辑非系统提供的关键字段的信息。
  • 系统提供的字段只能查看基本信息，不能做任何编辑或修改操作。
