# 运输费率设置

运输费率设置，即根据实际运输场景维护承运商提供的基于起始区域到目的区域和收费类型的报价信息，作为运输优化的重要考量指标供计费引擎所调用，保证输出的车次结果反应真实的运费水平，实现运输成本最小化，提高车次结果的可落地性。 运输费率涵盖：主收费项目、阶梯、区域三个子模块以及完整运输费率合同的维护功能。

# 1. 主收费项目

主收费项目定义了承运商按照什么条件进行运输收费，常见的运输收费项目包括：整车价、车公里单价、按重量计价、按体积计价等。

# 1.1 主收费项目信息

一个起始区域到目的区域，承运商的报价可以由多个收费项目组成。而每一个收费项目由查找因子和计费因子构成，附带一些收费项目的属性维护。

简而言之，查找因子表明了收费需要落入的阶梯，计费因子则表示在这个阶梯内按什么计费。目前系统支持通过下拉框从车次数、总件数、总距离、中途站点数、总单数、总站数、总体积、总重量、立方*公里、吨*公里等中选择因子。

案例：承运商针对上海到杭州的公路运输，报价4M2车，一车500块，超过200公里之后每公里10块

在以上报价模型内，收费项目有两个

  • 整车价：一车500块，查找因子是车次，计费因子也是车次
  • 车公里单价：每公里10块，查找因子公里，计费因子公里

需要注意的是，查找因子可以和计费因子不一样。例如重量在5吨以上的，每公里计费10块，那么查找因子是吨（重量），计费因子是公里。

除此之外，用户可以维护计费类型（系统计费/人工计费）、是否基于车型收费（后续费率维护时将根据此选项判断是否需要输入车型）、收费项目所属的公司部门等信息。

# 1.2 主收费项目信息（字段说明）

  • 基础信息

    • 收费项目代码：系统中收费项目的唯一代码

    • 收费项目描述：设置该收费项目的名称，方便辨别

    • 收费类型：常规收费表示一定会参与计算的费用，可选项目表示当符合条件出现时才会计算的费用，例如节假日送货费

    • 基于车型收费：是/否。表明此收费项目是否与车型有关

    • 查找因子：表示按何种因子判断计费落入的阶梯，例如“重量在5吨以的，每公里计费10块”的查找因子为总重量

    • 计费因子：表示按何种因子计算费用，例如“重量在5吨以的，每公里计费10块”的计费因子为总距离

    • 所属公司/部门：显示该主收费项目的所属公司及部门信息，不可编辑

# 2. 阶梯

收费阶梯是指计费过程中使用的阶梯，例如承运商报价0-100KM内每公里10块，100-200KM内每公里12块，那么其中的 0-100，100-200 即为收费阶梯。

与收费项目类似，维护的收费阶梯为主从结构，也将会被费率合同引用。

收费阶梯本身不含报价信息，但是包含阶梯内每个阶梯段按照什么类型计费，例如按单元计费、按固定金额计费和按百分比计费等。

# 2.1 收费阶梯信息

收费阶梯通过阶梯上下限、计费类型和是否剪切共同定义了计费逻辑。阶梯上下限约定了阶梯生效的区间条件，计费类型声明了计费方式，并通过是否剪切来处理多段计费的逻辑。

在上图所示的案例中，计算方法依据计费因子的数值落在哪个区域而定

  • 如果数值落小于100或大于20000，费率合同不生效，无费用
  • 如果落在100～200之间，则直接返回固定收费金额
  • 如果数值大于200且小于10000，并且由于不是剪切的，所以直接返回计算因子数据*单价
  • 如果数值大于10000且小于20000，且由于这一段是剪切的，但它的前一段不是剪切的，所以计费分为两部分：(计费因子数值 - 10000) * 第三段的单价 + 10000 * 第二段的单价。假如前一段设置为剪切，那么会重复以上步骤继续向上剪切，直到不剪切为止

# 2.2 收费阶梯信息（字段说明）

  • 基础信息

    • 阶梯代码：系统中阶梯的唯一代码
    • 阶梯描述：设置该阶梯的名称，方便辨别
    • 所属公司/部门：可设置阶梯信息的所属公司或部门信息；如未维护部门信息，则阶梯在公司内部全部门可用；如未维护公司信息，则取当前用户的所属公司及部门信息

  • 阶梯明细

    • 阶梯上/下限：表明阶梯的上下限，阶梯下限必须大于上限，下一段阶梯的起点必须大于上一阶梯的上限。允许阶梯不连续，
    • 计费类型：按单元计费表明落入该阶梯之后按照计费因子数量进行计费（乘以计费单价，在费率中维护）。按固定金额计费表明落入该阶梯之后，固定计算一笔费用（在费率中维护），按百分比计费表明落入该阶梯之后将在此收费项目之前的费用总和按百分比（在费率中维护）计算一笔费用
    • 是否剪切：表明该阶梯计费是否分段计算费用最后累加。是则按因子分段计算之后累加，否则不累加，按照最后落入的阶梯的单价计算总费用

# 3. 区域

系统内可以维护区域信息，标示一个地理区域，这个区域将会被费率合同以及运输算法引擎调用。这个区域可以是一个国家、一个省份、一个城市、一个行政区、一个邮编范围、一个自定义的区域划分（例如将上海划分为浦东浦西两个区域），或者直接绑定到某些具体的运输地点上。通过维护区域信息，系统可以实现不同区域不能拼载、某区域的运单内只能使用部分车型配送、不同起始/目的区域有不同的计费规则等业务场景。

# 3.1 区域信息维护

系统中的区域分为不同的图层，同一个地点在一个图层内只会属于一个区域

  • 目前支持限行图层、业务区域图层和费率图层3个
  • 维护了限行区域、业务区域的情况下，在系统配置模块打开“自动解析”功能，或在运输地点维护时打开“自动解析”功能时，任何新增的运输地点将会自动解析匹配一个限行区域/业务区域。此业务区域和限行区域信息将会被算法引擎识别，在拼载的时候考虑相关限行规则约束和业务分区约束。
  • 一个运输地点在业务图层和限行图层上只能属于一个区域
  • 只有限行图层区域支持维护限行规则

在区域明细的围栏页面窗口中，用户既可以手工新增、调整区域边界：

  • 开启多边形新增按钮后，通过鼠标单击多边形的边界点勾画边界，双击左键/单击右键后完成当前多边形绘制
  • 开启多边形调整按钮后，通过自由拖动多边形边界的线段中点可调整多边形的形状

也可以直接通过下拉框选择行政区划：

围栏编辑完成后，须点击区域详情页下方的“保存”按钮，系统才会自动解析围栏的经纬度信息并保存下来。

同一区域可以维护多条围栏，同一围栏可以维护多个多边形区域。

维护了多条区域信息后，可以在“运输费率-区域”列表页中勾选后点击页面下方的“围栏地图”按键以同时显示在同一地图中：

# 3.2 区域信息（字段说明）

  • 通用信息

    • 区域代码：系统中区域的唯一代码
    • 区域描述：设置该区域的名称，方便辨别
    • 所属公司/部门：显示该区域信息所属的公司及部门信息
    • 所属图层：设置该区域属于哪一个图层，共有业务/限行/费率三种图层供选择设置
    • 优先级：显示区域的优先级，同一图层的区域优先级越高（数字越小）越先被解析

  • 区域明细

  • 国家/省市/行政区域/邮编范围：通过设置不同的等级的区域地理范围来标示一个区域，例如可以设置一个上海市作为一个区域，也可以设置上海市杨浦区作为一个区域。同一个区域下的地理范围明细不可以有交叉

    • 运输地点：区域直接绑定到具体的地点上，将某些运输地点直接绑定为一个区域，例如起始仓库所在的区域，可以创建为一个只包含起始仓库地点的区域

  • 限行规则维护（只有限行区域可配置）

  • 限行类型：维护区域上的限行类型，通过下拉框形式选择。根据不同的限行类型，系统自动设置具体限行对象输入框的内容和格式。当维护了限行规则，优化引擎在计算车次时，会考虑相关限行要求，将符合限行的车次计算出来。目前支持的限行类型包括： - 车型限制通行：选择车型，表示设定的车型在指定工作日历上不可在此区域通行 - 尾号限行：选择尾号，可多选，表示设定车辆尾号在指定工作日历上不可在此区域通行 - 车牌限行：选择具体的车辆信息，表示设定的具体某台车辆在指定工作日历上不可在此区域通行 - 外地车牌限行：输入对于改区域来说属于外地的车牌，通过“非XXXX”来制定这个区域的“本地车牌”范围。对于京沪粤深，可以不用指定第二位的车牌（非京车牌即可）

    • 工作日历：设置限行的时间段。通过日历方式维护。每个限行必须维护时间段，表示在该时段内车型/车辆/尾号限制通行。

# 4. 费率

费率是承运商提供的基于起始区域到目的区域和收费类型的报价信息。费率的关键信息包括：承运商、合同头、提供的运输服务、起始区域、目的区域、收费类型、车型、阶梯和报价信息等。通过费率维护的信息将会被计费引擎所调用，用于校验车次是否有合法的计费信息。

# 4.1 费率信息维护

用户可以在“运输费率-费率”页面下，完成新增、编辑、删除费率操作。维护费率时需要选用系统中已维护的主收费项目、计费阶梯及费率区域数据。新增费率信息时，用户既可以选择在前端界面中维护，也可以通过模版批量上传。同一合同编号能够维护多条费率规则，在开启计费引擎时，算法在计算过程中会基于车型、起始/目的区域、有效期等因素，同时考虑所有匹配的费率合同来计算费用，寻求总费用最低的运输方案。

# 4.2 费率信息（字段说明）

  • 基础信息

    • 合同编号：表明该费率属于哪个运输合同

  • 承运商/承运商代码：表明该费率属于哪个承运商

    • 服务类型/服务类型代码：表明这个运输费率的服务类型，例如空运/陆运/整车/零担/海运/铁路运输等

  • 是否参与优化：表明当前费率是否需要被优化引擎考虑，设置为否，则该条费率不参与优化计算

    • 有效期：表明该费率的有效期限

  • 起始区域/目的区域：起始区域和目的区域唯一标示了一条线路

  • 收费信息

    • 收费项目代码：查询输入收费项目代码，决定了该条费率的查找因子和计费因子（查找因子和计费因子的详细说明参见“主收费项目”部分的说明

    • 阶梯代码：查询输入阶梯代码，费率明细将根据阶梯代码填充记录。

    • 车型代码：如果收费类型里指定为按车型收费，则必须填入此条费率的车型代码

    • 最大最小查找/计费因子数量：限制该条费率的最大最小查找/计费因子，如车次上的查找/计费因子数量大于这个限制，则不会被选取该条费率进行计算

    • 最大/最小收费金额：使用该条费率计算的金额的上下限，金额超过限制，以上下限最终计算结果

    • 基本收费金额：使用该条费率下的计费因子计算得到的金额将加上基础金额作为最终计算结果，受到最大最小收费金额的限制

  • 明细信息

    费率明细由收费阶梯和费用构成，收费阶梯参见上文“收费阶梯”部分的说明，而费用部分用户可以指定每个阶梯上的计费单价与基础金额。系统综合二者的信息后统一计算车次的真实费用。

    阶梯上下限、计费类型、是否剪切，请参照“阶梯”部分的说明。

    基础金额表示：查找因子落在此阶梯内是，固定计算的一笔费用。
